# Plan: Training Provenance & Metadata Tracking

## Problem

When you train a model, there's no record of:
- What data was used (which .npy file, how many tokens, what languages)
- What filter settings were applied (conservative/strict, rejection rate)
- What config was used (model size, learning rate, batch size)
- What hardware it ran on (GPU, VRAM, precision)
- How long it took, final loss, checkpoints saved

This makes it impossible to reproduce results or compare training runs.

## Solution: `training_manifest.yaml`

Every training run produces a manifest file saved alongside the checkpoint.
Every data preparation run produces a manifest saved alongside the .npy file.

## Implementation

### Files to modify:

1. **`src/cola_coder/data/preprocess.py`**
   - After `tokenize_and_chunk()` completes, write manifest YAML
   - Include filter stats, timing, source info
   - Save as `{output_name}.manifest.yaml` next to the `.npy` file

2. **`src/cola_coder/training/checkpoint.py`**
   - `save_checkpoint()` creates/updates `training_manifest.yaml` in checkpoint output dir
   - Includes full config, data provenance (read from data manifest), loss history
   - Updated at every checkpoint, not just first save

3. **`src/cola_coder/training/trainer.py`**
   - Pass data manifest path to trainer
   - Track loss history array across steps
   - Compute epochs_completed, tokens_seen

4. **`scripts/prepare_data.py`**
   - Gather all metadata (filter stats, timing, config values)
   - Pass to preprocess module for manifest creation

5. **`scripts/train.py`**
   - Load data manifest from `.manifest.yaml` next to the training data file
   - Pass to trainer for inclusion in training manifest

## Why This Matters

- **Reproducibility**: Anyone can recreate your exact training run
- **Comparison**: Compare two models by diffing their manifests
- **Debugging**: When a model is bad, trace back to data quality or config issue
- **Compliance**: Industry trend — model cards, data cards are becoming required
- **Multi-agent**: When training specialist models, manifests track which data each saw
