# 64 - Real-World Evaluation (GitHub Issues → Fix → Compile)

## Overview

Pull real "good first issue" TypeScript bugs from GitHub, have Cola-Coder generate fixes, apply them to the repository, and run `tsc` to check if the fix compiles. This provides a ground-truth, real-world evaluation signal rather than synthetic benchmarks.

**Feature flag:** `--enable-real-world-eval` (off by default; requires GitHub token and disk space)

---

## Motivation

Synthetic benchmarks measure what you built them to measure. Real-world GitHub issues measure whether the model can actually help with the code that exists in the wild. The gap between synthetic pass rates and real-world usefulness is well-documented in ML literature.

Specific benefits for Cola-Coder:
- Issues on real repos use real import structures, real types, real project conventions
- The compile check (`tsc`) is an objective, deterministic pass/fail signal
- A model that improves on this metric is genuinely becoming more useful
- Real issues are continuously generated by the open-source community, providing an ever-fresh evaluation set

**The challenge**: setting up arbitrary TypeScript repos (installing `node_modules`, resolving workspace deps) is fragile. The plan provides a staged approach from simplest to most complete.

---

## Architecture / Design

### Evaluation Pipeline Stages

```
Stage 1: Standalone compile check (simplest)
  Generate code snippet → write to temp .ts file →
  run tsc with --noEmit --strict → pass/fail

Stage 2: Single-file repo context
  Download issue's file → inject generated fix →
  run tsc on that file with inlined types → pass/fail

Stage 3: Full repo (complex, optional)
  Clone repo → npm install → apply fix → tsc → pass/fail
```

**Default implementation: Stage 2.** Stage 1 is too easy (no cross-file types). Stage 3 is too fragile (npm install failures, large repos). Stage 2 hits the sweet spot.

### Issue → Prompt Conversion

```python
def issue_to_prompt(issue: dict, file_context: str = "") -> str:
    """Convert a GitHub issue to a model prompt."""
    body = issue["body"] or ""

    # Extract code blocks from issue
    code_blocks = re.findall(r'```(?:typescript|ts|javascript|js)?\n(.*?)\n```',
                              body, re.DOTALL)

    prompt = f"""// Fix the following TypeScript issue:
// {issue['title']}
//
// Description: {body[:500]}
"""
    if code_blocks:
        prompt += f"\n// Problematic code:\n{code_blocks[0][:500]}\n\n"
    if file_context:
        prompt += f"// File context:\n{file_context[:300]}\n\n"

    prompt += "// Fixed code:\n"
    return prompt
```

## Implementation Steps

### Step 1: GitHub Issue Fetcher (`eval/github_fetcher.py`)

```python
import os
import time
import requests
from dataclasses import dataclass
from pathlib import Path
import json

GITHUB_API = "https://api.github.com"

@dataclass
class IssueRecord:
    repo: str
    issue_number: int
    title: str
    body: str
    code_blocks: list[str]
    file_url: str | None        # URL to the referenced file, if parseable
    raw_url: str | None         # Raw content URL

class GitHubIssueFetcher:
    def __init__(self, token: str, cache_dir: Path):
        self.token = token
        self.cache_dir = cache_dir
        self.cache_dir.mkdir(parents=True, exist_ok=True)
        self.session = requests.Session()
        self.session.headers.update({
            "Authorization": f"token {token}",
            "Accept": "application/vnd.github.v3+json",
        })

    def fetch_repos(self, max_repos: int = 20) -> list[dict]:
        """Fetch TypeScript repos with good first issues."""
        cache_file = self.cache_dir / "repos.json"
        if cache_file.exists():
            return json.loads(cache_file.read_text())

        params = {
            "q": "language:TypeScript stars:>100 good-first-issues:>5",
            "sort": "stars",
            "order": "desc",
            "per_page": max_repos,
        }
        resp = self.session.get(f"{GITHUB_API}/search/repositories", params=params)
        resp.raise_for_status()
        repos = resp.json()["items"]
        cache_file.write_text(json.dumps(repos, indent=2))
        return repos

    def fetch_issues(self, repo_full_name: str, max_issues: int = 10) -> list[IssueRecord]:
        """Fetch good first issues from a repo."""
        cache_file = self.cache_dir / f"{repo_full_name.replace('/', '_')}_issues.json"
        if cache_file.exists():
            raw = json.loads(cache_file.read_text())
            return [IssueRecord(**r) for r in raw]

        params = {
            "labels": "good first issue,bug",
            "state": "open",
            "per_page": max_issues,
        }
        resp = self.session.get(
            f"{GITHUB_API}/repos/{repo_full_name}/issues", params=params
        )
        if resp.status_code == 404:
            return []
        resp.raise_for_status()

        records = []
        for issue in resp.json():
            if "pull_request" in issue:
                continue  # Skip PRs
            body = issue.get("body") or ""
            code_blocks = re.findall(
                r'```(?:typescript|ts|javascript|js)?\n(.*?)\n```',
                body, re.DOTALL
            )
            if not code_blocks:
                continue  # Skip issues without code

            records.append(IssueRecord(
                repo=repo_full_name,
                issue_number=issue["number"],
                title=issue["title"],
                body=body[:2000],
                code_blocks=code_blocks,
                file_url=None,
                raw_url=None,
            ))
            time.sleep(0.1)  # Rate limit courtesy

        cache_file.write_text(json.dumps(
            [r.__dict__ for r in records], indent=2
        ))
        return records
```

### Step 2: Fix Generator and Compiler (`eval/real_world_eval.py`)

```python
import subprocess
import tempfile
import os
from pathlib import Path

class RealWorldEvaluator:
    def __init__(self, model, tokenizer, config: dict):
        self.model = model
        self.tokenizer = tokenizer
        self.tsc_timeout = config.get("tsc_timeout", 15)
        self.max_gen_tokens = config.get("max_gen_tokens", 512)

    def generate_fix(self, prompt: str) -> str:
        """Generate a fix using the model."""
        tokens = self.tokenizer.encode(prompt)
        input_ids = torch.tensor([tokens]).to(self.model.device)

        with torch.no_grad():
            output_ids = self.model.generate(
                input_ids,
                max_new_tokens=self.max_gen_tokens,
                temperature=0.7,
                do_sample=True,
            )

        generated = output_ids[0][len(tokens):]
        return self.tokenizer.decode(generated.tolist())

    def check_compiles_standalone(self, code: str) -> tuple[bool, str]:
        """Write code to temp file and run tsc --noEmit --strict."""
        with tempfile.TemporaryDirectory() as tmpdir:
            ts_file = Path(tmpdir) / "fix.ts"
            ts_file.write_text(code)

            # Minimal tsconfig
            tsconfig = Path(tmpdir) / "tsconfig.json"
            tsconfig.write_text('{"compilerOptions": {"strict": true, "noEmit": true}}')

            try:
                result = subprocess.run(
                    ["tsc", "--project", str(tsconfig)],
                    capture_output=True,
                    text=True,
                    timeout=self.tsc_timeout,
                    cwd=tmpdir,
                )
                passed = result.returncode == 0
                errors = result.stdout + result.stderr
                return passed, errors
            except subprocess.TimeoutExpired:
                return False, "tsc timeout"
            except FileNotFoundError:
                return False, "tsc not found in PATH"

    def evaluate_issue(self, issue: IssueRecord) -> dict:
        prompt = issue_to_prompt(issue)
        fix = self.generate_fix(prompt)
        passed, errors = self.check_compiles_standalone(fix)

        # Count error types
        error_types = []
        if not passed:
            for line in errors.split("\n"):
                match = re.search(r'TS(\d+)', line)
                if match:
                    error_types.append(f"TS{match.group(1)}")

        return {
            "repo": issue.repo,
            "issue_number": issue.issue_number,
            "title": issue.title,
            "passed": passed,
            "error_count": len(error_types),
            "error_types": list(set(error_types)),
            "generated_tokens": len(self.tokenizer.encode(fix)),
        }

    def run_evaluation(self, issues: list[IssueRecord]) -> dict:
        results = []
        for issue in issues:
            result = self.evaluate_issue(issue)
            results.append(result)

        passed = sum(1 for r in results if r["passed"])
        total = len(results)

        # Aggregate error type frequency
        all_errors = []
        for r in results:
            all_errors.extend(r["error_types"])
        from collections import Counter
        error_freq = Counter(all_errors).most_common(10)

        return {
            "total": total,
            "passed": passed,
            "compile_success_rate": passed / total if total > 0 else 0.0,
            "common_errors": error_freq,
            "results": results,
        }
```

### Step 3: CLI Command (`eval/cmd_real_world.py`)

```python
# cola-coder eval real-world \
#   --checkpoint checkpoints/step-5000.safetensors \
#   --max-issues 50 \
#   --github-token $GITHUB_TOKEN \
#   --output eval_results/real_world_step5000.json

def run_real_world_eval(args):
    fetcher = GitHubIssueFetcher(
        token=args.github_token,
        cache_dir=Path(".eval_cache/github"),
    )
    repos = fetcher.fetch_repos(max_repos=10)
    all_issues = []
    for repo in repos:
        issues = fetcher.fetch_issues(repo["full_name"], max_issues=5)
        all_issues.extend(issues)
        if len(all_issues) >= args.max_issues:
            break

    model, tokenizer = load_checkpoint(args.checkpoint)
    evaluator = RealWorldEvaluator(model, tokenizer, config=vars(args))
    summary = evaluator.run_evaluation(all_issues[:args.max_issues])

    output_path = Path(args.output)
    output_path.parent.mkdir(parents=True, exist_ok=True)
    output_path.write_text(json.dumps(summary, indent=2))

    console = Console()
    console.print(f"\n[bold]Real-World Eval Results[/]")
    console.print(f"  Issues evaluated: {summary['total']}")
    console.print(f"  Compile success:  [green]{summary['compile_success_rate']*100:.1f}%[/]")
    console.print(f"  Top errors: {summary['common_errors'][:3]}")
```

## Testing Strategy

1. **Fetcher caching test**: call `fetch_repos` twice, assert second call returns cached result without making HTTP requests (mock the session).
2. **Compile check test**: write a known valid TypeScript snippet, assert `check_compiles_standalone` returns `(True, "")`. Write a snippet with a deliberate type error, assert `(False, error_msg)`.
3. **Timeout test**: write a TypeScript file that causes tsc to hang (infinite loop at type-check level is impossible, but test subprocess timeout with a shell script mock).
4. **Integration test**: run eval on 5 cached issue fixtures, assert output JSON has correct schema.
5. **Rate limit test**: mock GitHub API to return 403, assert graceful error with message.

## Performance Considerations

- GitHub API: 5000 requests/hour with token. Fetching 50 issues from 10 repos requires ~110 requests. Cache aggressively.
- `tsc` invocation is 0.5-2s per file (JIT warm-up dominates). 50 issues = up to 100s. Run evaluations in a subprocess pool if targeting 200+ issues.
- Each evaluation requires a model forward pass. At batch size 1 this is slow. Pre-generate all fixes first, then batch-compile.
- Disk space: temp directories are cleaned up by `tempfile.TemporaryDirectory`. Cache directory can grow large; implement TTL-based pruning.

## 2026 Best Practices

- **Cache everything from GitHub**: API rate limits and repo churn make uncached evaluation non-reproducible. Store issues as JSON with a timestamp; re-fetch only when TTL expires.
- **Separate generation from evaluation**: generate all model outputs first (batch them for efficiency), then run compile checks. This allows re-evaluating without regenerating.
- **Error taxonomy matters**: TS error codes are a rich signal. Track whether the model is improving on specific error types (TS2339 property doesn't exist, TS2345 argument type mismatch). This guides training interventions.
- **Standalone compile as minimum bar**: even if Stage 2 is implemented, always include standalone compile as a fast sanity check. It catches the worst failures in <2s.
- **Disclosure**: when reporting eval results, always note "compile success on standalone snippets, not full project context" to avoid overstating capability.
